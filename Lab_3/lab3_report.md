# 2. Теоретическая часть

- 2.1 Модели хранения данных

В рамках лабораторной работы были изучены основные модели хранения данных:

1. Блочное хранилище

Используется для виртуальных машин и баз данных. Данные хранятся в виде блоков фиксированного размера. Пример: Persistent Disk.

2. Файловое хранилище

Предоставляет доступ к данным через файловую систему. Используется для совместной работы нескольких сервисов.

3. Объектное хранилище

Данные хранятся в виде объектов (файл + метаданные + уникальный идентификатор).
Google Cloud Storage относится к объектному хранилищу.

# 3. Практическая часть

## 3.1 Создание bucket

В выбранном проекте был создан bucket со следующими параметрами:

1. Название: lab3-kazar-bucket

2. Location type: Multi-region

3. Location: US

4. Storage class: Standard

5. Public access prevention: Initially enabled

После создания bucket был выполнен переход в раздел Objects для загрузки данных

## 3.2 Загрузка изображений

В созданный bucket были загружены 3 изображения:

- 1.webp

- 2.jpeg

- 3.webp

Загрузка выполнялась через кнопку Upload в интерфейсе Cloud Storage.

## 3.3 Создание папки и перемещение файлов

В bucket была создана папка:
```
images/
```
После этого все загруженные файлы были перемещены внутрь данной папки с помощью функции Move.
Это демонстрирует логическую структуру хранения данных в объектном хранилище.

## 3.4 Настройка публичного доступа

Изначально bucket имел включённую настройку Public Access Prevention, что запрещало добавление публичных прав доступа.

Для настройки публичного доступа были выполнены следующие действия:

1. Перейдено в раздел Protection

2. Отключена настройка Public Access Prevention

3. Перейдено в раздел Permissions

4. Добавлен принципал:
```
allUsers
```
5. Назначена роль:
```
Storage Object Viewer
```
После сохранения настроек объекты стали доступны публично через прямую ссылку

## 3.5 Генерация публичной ссылки
Для одного из файлов была создана публичная ссылка через контекстное меню объекта
```
https://storage.googleapis.com/lab3-kazar-bucket/images/1.webp
```
Файл успешно открывается в браузере без авторизации, что подтверждает корректную настройку публичного доступа

## 3.6 Удаление ресурсов

После завершения лабораторной работы были выполнены следующие действия:

1. Удалены все объекты внутри bucket

2. Удалён bucket lab3-kazar-bucket

3. Проверено отсутствие активных ресурсов
